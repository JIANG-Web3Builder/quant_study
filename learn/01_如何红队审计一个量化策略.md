# 如何红队审计一个量化策略

## 1. 红队审计的基本立场

红队审计不是优化策略，而是攻击策略。你要暂时站在反方：

```text
假设这个策略是假的。
我要找出它为什么看起来有效。
```

这和普通复盘不同。普通复盘经常问“哪里可以改进”，红队审计先问“它是不是根本不成立”。

## 2. 策略审计总图

```text
原始策略结果
  -> 数据泄露攻击
  -> 标签可交易性攻击
  -> 统计显著性攻击
  -> 多重检验攻击
  -> 风格/行业/市值暴露攻击
  -> 样本外攻击
  -> 成本与容量攻击
  -> 组合工程攻击
  -> 实盘监控攻击
  -> 最终判决
```

每一层攻击都可能杀死策略。

## 3. 第一轮攻击：这个结果是不是数据问题？

优先检查数据，因为数据问题不可用漂亮统计修复。

### 攻击问题

- 财报数据是否按公告日生效？
- 指数成分是否使用历史成分？
- 行业分类是否使用历史分类？
- 是否使用未来修订后的数据？
- 是否用 t 日收盘后信号在 t 日收盘成交？
- 是否删除了后来退市或长期停牌的股票？
- 是否把无法成交的涨跌停股票当作可成交？

### 通过标准

策略必须能明确说明每个字段在历史日期上的可见时间。

### 失败标准

只要核心信号依赖未来不可见数据，策略直接死亡。不要继续做样本外，不要继续调参。

## 4. 第二轮攻击：这个显著性是不是假的？

很多策略不是数据泄露，而是统计上被高估。

### 攻击问题

- 一共试了多少个因子或参数？
- 失败实验有没有记录？
- t 值是否考虑标签重叠？
- 收益是否集中在少数几个月？
- IC 是否有明显自相关？
- 是否只展示了最好的一组参数？
- 是否进行了样本选择？

### 必做实验

- 参数扰动测试。
- 随机标签测试。
- 时间块 bootstrap。
- 分年份显著性。
- Newey-West t 值。
- 成本翻倍压力测试。

### 通过标准

策略不需要每个测试都完美，但不能只依靠单一最优参数、单一市场阶段或单一统计口径成立。

## 5. 第三轮攻击：收益是不是风险暴露？

很多所谓 alpha 只是风险暴露。

### 常见伪来源

- 小市值。
- 低流动性。
- 高波动。
- 行业轮动。
- Beta。
- 动量或反转。
- 低价股。
- 困境股风险补偿。

### 必做实验

- 行业中性前后对比。
- 市值中性前后对比。
- 分市值组 IC。
- 分流动性组 IC。
- 分行业 IC。
- Barra 风格暴露回归。
- 与常见因子收益相关性。

### 通过标准

中性化后可以下降，但不能完全消失且无法解释。如果中性化后死亡，要诚实承认它是风险暴露或风格交易。

## 6. 第四轮攻击：样本外是不是被污染？

很多人以为自己做了样本外，其实测试集已经被间接用过。

### 污染方式

- 看过测试集后改因子。
- 看过测试集后改参数。
- 用全样本筛选因子。
- 用全样本决定行业处理方式。
- 用全样本选择持有期。
- 用全样本决定股票池过滤规则。

### 必做实验

- 固定研究规则后，找新的时间段重测。
- Walk-forward。
- Purged CV。
- 不同市场状态重测。
- 不同股票池重测。

### 通过标准

样本外可以变弱，但不能完全反向，且衰减原因要能解释。

## 7. 第五轮攻击：能不能真实交易？

纸面收益经常死在交易层。

### 攻击问题

- 换手率多少？
- 成本假设是否足够保守？
- 低流动性股票贡献多少收益？
- 涨跌停是否影响买卖？
- 停牌是否导致持仓偏离？
- 资金规模扩大后收益怎么变？
- 交易日是否过于集中？

### 必做实验

- 成本 1 倍、2 倍、3 倍压力测试。
- 不同资金规模容量测试。
- 剔除最低流动性股票。
- 延迟一天成交测试。
- 限制参与率测试。

### 通过标准

策略在合理成本和容量下仍有剩余收益。如果收益全部来自无法成交的小票，策略死亡。

## 8. 红队审计输出

每次审计输出必须包括：

```text
原始结论：
最强证据：
最大漏洞：
已通过攻击：
未通过攻击：
需要补做实验：
最终判决：
```

最终判决不能含糊。

## 9. 训练任务

选择一个你过去最满意的策略，用本文框架写一份红队审计。不要先改策略，只审计原始版本。

重点写：

- 哪一轮攻击最致命？
- 如果策略死亡，死亡原因是什么？
- 如果策略没死，剩下的 alpha 证据是什么？
