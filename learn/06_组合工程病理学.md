# 组合工程病理学

## 1. 为什么组合会把 alpha 弄坏

一个因子单独检验有效，不代表组合后有效。组合工程会引入新的问题：

- alpha 尺度错误。
- 风险模型估计误差。
- 协方差矩阵病态。
- 约束互相冲突。
- 优化器放大噪声。
- 成本模型低估。
- 权重集中。
- 暴露漂移。

组合不是因子结果的自然延伸，而是另一个研究对象。

## 2. 病例一：把 score 当预期收益

### 症状

优化器给出极端持仓，集中买入少数高分股票。

### 病因

alpha score 只是横截面排序分数，不是真实预期收益。如果把 z-score 直接当收益率，优化器会过度相信分数差异。

### 诊断

- 检查 alpha 输入单位。
- 检查单票主动权重。
- 检查前十大持仓贡献。
- 缩小 alpha 尺度后重跑优化。

### 修复

- 对 alpha 做 shrinkage。
- 使用 rank score。
- 限制单票权重。
- 提高风险惩罚。
- 使用分层组合做对照。

## 3. 病例二：优化器放大协方差误差

### 症状

组合理论风险很低，实际波动很高。

### 病因

协方差矩阵估计不稳定，优化器找到估计误差中的“漏洞”，构建了看似低风险、实际高风险的组合。

### 诊断

- 比较预测风险和实现风险。
- 检查协方差矩阵条件数。
- 检查风险贡献是否集中。
- 换一个风险模型重跑。
- 对协方差做 shrinkage 后对比。

### 修复

- 使用因子风险模型。
- 对协方差矩阵做 shrinkage。
- 设置风险贡献上限。
- 限制行业和风格暴露。
- 降低优化频率。

## 4. 病例三：约束吃掉 alpha

### 症状

单因子分组收益很好，但优化组合表现一般。

### 病因

alpha 主要存在于某些行业、市值段或低流动性股票中，但组合约束禁止表达这些暴露。

### 诊断

- 比较无约束组合和有约束组合。
- 逐个加入约束，观察收益下降。
- 计算每个约束的 shadow cost。
- 查看约束贴边频率。

### 修复

- 承认 alpha 依赖某种暴露。
- 调整策略定位，而不是硬称纯 alpha。
- 只在适用组合中使用。
- 降低约束强度或改为风险预算。

## 5. 病例四：组合收益来自少数股票

### 症状

总体收益很好，但去掉几只股票后表现显著下降。

### 病因

组合有效性来自偶然重仓的少数股票，不是稳定横截面规律。

### 诊断

- 单票收益贡献排名。
- 去掉 top contributor 重测。
- 持仓集中度。
- Herfindahl 指数。
- 前十大持仓权重。

### 修复

- 降低单票上限。
- 提高持仓数量。
- 使用行业内分散。
- 限制单票 alpha 贡献。

## 6. 病例五：换手约束导致信号过期

### 症状

加换手约束后收益下降很多。

### 病因

信号半衰期短，但组合为了控制换手无法及时调整，导致持有过期 alpha。

### 诊断

- 计算 alpha decay。
- 比较不同换手上限。
- 查看未交易目标权重。
- 检查持仓 score 分布。

### 修复

- 只使用半衰期更长的信号。
- 降低调仓频率。
- 对交易设优先级。
- 对高置信信号允许更高换手。

## 8. 通过和失败标准

### 通过

- 组合收益不依赖少数股票。
- 预测风险和实现风险差距可解释。
- 约束没有完全吃掉 alpha。
- 风险暴露符合策略定位。
- 成本和换手可接受。

### 失败

- 优化后收益主要来自约束漏洞。
- 风险模型严重低估风险。
- alpha 被约束后完全消失。
- 组合高度集中且无解释。
- 实现持仓和目标持仓长期偏离。
