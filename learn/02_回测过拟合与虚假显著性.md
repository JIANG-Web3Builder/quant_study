# 回测过拟合与虚假显著性

## 1. 问题场景

你有一个策略：

```text
样本期：2015-2024
年化收益：28%
夏普：2.1
最大回撤：12%
RankIC 均值：0.045
t 值：4.3
```

普通结论可能是：策略很好。

红队结论要先问：

```text
这个结果是在多少次尝试后挑出来的？
```

如果你试了 300 个因子、20 个参数、5 个股票池、4 种中性化方法，最后留下这个结果，那么它的显著性远低于表面 t 值。

## 2. 为什么普通 t 值经常高估

普通 t 值默认：

- 样本独立。
- 分布相对稳定。
- 只做了一次检验。
- 参数不是事后挑选。

量化研究通常不满足这些假设。

### 标签重叠

如果每天预测未来 20 日收益，那么相邻样本共享大部分收益窗口。

```text
t 日标签：t+1 到 t+20
t+1 日标签：t+2 到 t+21
```

这会导致 IC 序列自相关，有效样本数小于表面交易日数量。

### 截面相关

同一行业、同一风格股票收益相关。横截面股票数很多，不等于独立样本很多。

### 多次试验

你不是只测试了一个策略，而是测试了很多候选策略。最高的那个结果天然有选择偏差。

## 3. 多重检验的残酷直觉

假设所有因子都无效，你测试 1000 个因子，在 5% 显著性水平下，理论上会出现约 50 个“显著”结果。

所以你看到一个 p-value 小于 0.05 的因子时，不应该问：

```text
它是否显著？
```

而应该问：

```text
在我测试过的所有东西里，它仍然显著吗？
```

## 4. 必做审计实验

### 4.1 实验次数盘点

记录：

- 测过多少因子。
- 测过多少参数。
- 测过多少股票池。
- 测过多少持有期。
- 测过多少中性化方式。
- 测过多少成本假设。

如果没有记录，默认存在 data mining 风险。

### 4.2 参数扰动测试

如果最优参数是：

```text
lookback = 37
holding = 13
```

你要测试附近参数：

```text
lookback = 20, 30, 40, 60
holding = 5, 10, 20
```

通过标准：

```text
不是只有一个孤立参数点有效。
```

失败信号：

```text
最优点像针尖一样突出，周围全部失效。
```

### 4.3 时间块扰动

把样本切成多个连续时间块，观察每段表现。

失败信号：

- 80% 收益来自一个年份。
- 某个极端行情贡献大部分收益。
- 去掉最好一年后策略失效。

### 4.4 随机化测试

做几种 sanity check：

- 打乱因子日期。
- 打乱股票标签。
- 随机生成同分布因子。
- 用滞后标签替代真实标签。

如果随机因子也能产生类似表现，说明研究流程本身可能过拟合。

### 4.5 Newey-West 或保守 t 值

对于重叠标签，不要只报普通 t 值。至少报告：

- 普通 t 值。
- 自相关调整后 t 值。
- 有效样本数估计。
- 使用 lag 的理由。

## 5. Deflated Sharpe 的直觉

如果你只测试一个策略，夏普 2 可能很强。

如果你测试了 1000 个策略，再挑最高夏普，夏普 2 可能只是“最大噪声”。

Deflated Sharpe Ratio 的思想是：考虑试验次数、样本长度、收益分布偏度峰度后，重新评估夏普是否真的显著。

你不一定一开始就要精确实现 DSR，但必须理解它提醒的事：

```text
回测表现要根据研究过程打折。
```

## 6. 红队判决标准

### 直接死亡

- 测试集被反复用于调参。
- 参数孤岛明显。
- 收益集中在少数极端日期且无法解释。
- 随机化测试也能产生类似结果。

### 降级为假设

- 有一定统计证据，但实验次数未知。
- 样本外较弱但未反向。
- 参数稳定性一般。

### 继续研究

- 有完整失败记录。
- 参数扰动后仍有效。
- 分时期表现不过度集中。
- 调整 t 值后仍有边际显著性。
- 样本外没有完全坍塌。
