# 机器学习量化反模式

## 1. 为什么 ML 量化更需要红队

机器学习模型会给人一种错觉：

```text
模型更复杂，所以更高级。
```

但在金融数据里，复杂模型更容易把泄露、噪声、缺失模式、样本偏差学得很好。

ML 量化红队重点不是问模型用了什么，而是问：

```text
模型到底学到了什么？
```

## 2. 反模式一：随机 K-fold

### 表面做法

把所有样本随机分成训练集和测试集。

### 为什么错

金融样本有时间结构。随机切分会让未来市场状态、相邻标签、相似股票信息泄露到训练集。

### 修复

- Walk-forward。
- 时间块切分。
- Purged CV。
- Embargo。

### 死亡信号

随机 K-fold 表现很好，时间切分表现明显下降。

## 3. 反模式二：全样本特征筛选

### 表面做法

先用全样本筛选高 IC 特征，再训练模型。

### 为什么错

测试集信息已经通过特征筛选进入训练过程。

### 修复

每个训练窗口内独立完成：

- 缺失处理。
- 特征筛选。
- 标准化。
- 中性化。
- 模型训练。

然后只在未来窗口测试。

## 4. 反模式三：模型指标提升但投资指标没提升

### 表面现象

MSE 降低，AUC 提高。

### 真实问题

投资组合关心的是排序、尾部识别、成本后收益、风险暴露，不一定关心平均误差。

### 必查指标

- RankIC。
- Top quantile return。
- Long-short return。
- 成本后收益。
- 换手率。
- 风格暴露。
- 分组单调性。

如果 ML 指标提升但这些指标不提升，模型没有投资价值。

## 5. 反模式四：模型学到了市值和流动性

### 表面现象

模型预测能力强，特征重要性分散。

### 可能真相

模型通过大量 proxy 学到了：

- 市值。
- 成交额。
- 波动率。
- 上市时间。
- 行业。
- 数据覆盖率。

### 红队实验

- 删除市值和流动性相关特征。
- 对预测 score 做市值、行业、流动性中性化。
- 分市值组测试。
- 分流动性组测试。
- 检查预测值和常见风格因子的相关性。

## 6. 反模式五：缺失值本身泄露

### 表面现象

填充缺失值后模型效果很好。

### 可能真相

缺失本身代表公司类型或数据供应商覆盖偏差。

### 红队实验

- 只用缺失哑变量训练模型。
- 按数据覆盖率分组测试。
- 删除覆盖变化大的字段。
- 检查不同年份缺失率变化。

如果只用缺失模式就能赚钱，要非常小心。

## 7. 反模式六：超参数调出了测试集

### 表面做法

反复调模型，直到测试集表现好。

### 为什么错

测试集被当成验证集使用，最终结果没有外推意义。

### 修复

- 训练集训练。
- 验证集调参。
- 测试集只看一次。
- 最后进入模拟实盘。

如果已经反复看过测试集，就承认它已污染，不要再把它叫测试集。

## 8. 反模式七：模型无法监控

复杂模型上线后，如果没有监控，很难知道它何时失效。

必须监控：

- 特征分布漂移。
- 预测分布漂移。
- 预测 rank 稳定性。
- 分组收益。
- 滚动 IC。
- 风格暴露漂移。
- 换手异常。
- 成交成本异常。

模型不是训练完成就结束，而是进入衰减监控。

## 9. ML 策略红队审计表

```text
标签是否可交易：
特征是否 point-in-time：
是否随机切分：
是否全样本特征筛选：
验证集是否被反复使用：
测试集是否只看一次：
ML 指标提升：
投资指标提升：
预测 score 的风格暴露：
删除疑似 proxy 后表现：
模型监控指标：
最终判决：
```

## 10. 判决标准

### 通过

- 时间切分后仍有投资指标提升。
- 删除明显 proxy 后仍有边际价值。
- 模型 score 可解释或可监控。
- 成本后收益优于简单线性基准。

### 失败

- 随机切分有效，时间切分无效。
- 投资指标无提升。
- 主要学习缺失模式或流动性。
- 测试集被反复调参污染。
- 无法监控模型退化。
