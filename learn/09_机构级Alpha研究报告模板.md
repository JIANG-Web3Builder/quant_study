# 机构级 Alpha 研究报告模板

## 1. 报告目标

这份报告不是为了展示策略有多好，而是为了让别人能判断：

```text
这个策略是否值得继续投入研究、组合测试或模拟实盘？
```

一份机构级报告必须允许别人攻击它。隐藏漏洞的报告没有价值。

## 2. 一页摘要

```text
策略名称：
研究假设：
交易标的：
样本期：
调仓频率：
持有期：
原始表现：
样本外表现：
成本后表现：
换手率：
估计容量：
最大风险：
最终建议：
```

最终建议只能选：

- `死亡`
- `保留假设`
- `继续研究`
- `进入组合测试`
- `进入模拟实盘`

## 3. 研究假设

写清楚：

```text
谁在犯错？
谁受约束？
为什么价格会偏离？
为什么偏离不会立刻消失？
什么市场环境下它应该更有效？
什么情况下它应该失效？
```

如果你写不出来，就说明它是统计型信号，需要更严格的验证。

## 4. 数据与泄露审计

```text
核心字段：
数据源：
字段可见时间：
是否会修订：
是否按公告日处理：
是否使用历史成分股：
是否处理退市：
是否处理停牌：
是否处理涨跌停：
泄露风险等级：
```

必须附一句明确结论：

```text
我认为当前数据审计是否足以进入下一步，原因是：
```

```text

```

必须写明：

```text

```

## 7. 因子尸检

```text
市值中性前后：
行业中性前后：
流动性分组：
分行业表现：
分市值组表现：
分市场状态表现：

```

必须给因子重新命名。

例如：

```text
原名：高质量成长
尸检后命名：小市值盈利改善暴露
```


```text

```



## 10. 执行与容量

```text
单票最大参与率：

估计容量：

```

必须回答：

```text
当前策略在多大资金规模下仍值得交易？
```

## 11. 失败与限制

这一节必须认真写。至少包括：

- 已知弱点。
- 未完成实验。
- 最担心的风险。
- 哪些结果可能被高估。
- 哪些市场环境可能失效。
- 哪些数据问题尚未完全排除。

没有限制的报告通常不可信。

## 12. 最终判决

```text
判决：
理由：
下一步实验：
下线或停止条件：
```

示例：

```text
判决：继续研究
理由：样本外仍有弱正 IC，市值中性后未完全消失，但容量较小，且成本敏感。
下一步：只在高流动性股票池重构组合，并做 3 个月模拟实盘。
停止条件：模拟实盘滚动 60 日 RankIC < 0 且成本后多空收益为负。
```
