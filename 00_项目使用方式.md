# 项目使用方式

## 1. 这不是知识库，而是审计框架

这个项目的目标不是收集量化知识点，而是训练一种研究姿势：

```text
把每一个策略当作嫌疑人，而不是当作作品。
```

如果你已经做过很多项目，最有效的进阶方式不是继续找新因子，而是把旧项目拿出来做“尸检”。很多旧项目在基础回测层面看起来不错，但一旦经过数据泄露、统计显著性、样本外、成本、容量、风险暴露审计，就会暴露真实质量。

## 2. 每个策略都要经历五道门

### 第一门：数据门

问题：

- 数据是否 point-in-time？
- 财报、公告、指数成分、行业分类是否存在未来修订？
- 标签和特征是否时间错位？

如果数据门过不了，策略直接死亡，不进入下一阶段。

### 第二门：统计门

问题：

- 显著性是否被高估？
- 是否存在多重检验？
- 样本是否独立？
- 收益是否集中在少数时间段？

如果统计门过不了，策略最多只能作为研究假设，不能作为 alpha。

### 第三门：解释门

问题：

- 收益来自什么机制？
- 是否只是市值、行业、流动性、波动率、动量等风险暴露？
- 为什么这个机会不会被套利掉？

如果解释门过不了，策略需要更强样本外证据。

### 第四门：组合门

问题：

- alpha 转成权重后是否还存在？
- 约束是否吃掉 alpha？
- 风险模型是否漏掉关键暴露？
- 策略和已有策略是否高度相关？

如果组合门过不了，说明信号可能有研究价值，但没有投资价值。

### 第五门：交易门

问题：

- 成本后是否还有效？
- 容量是多少？
- 涨跌停、停牌、滑点、冲击成本是否处理？
- 实盘中是否能监控和下线？

如果交易门过不了，策略只能停留在纸面。

## 3. 使用流程

拿一个旧项目，按下面顺序写审计记录：

```text
策略摘要
-> 原始结果
-> 数据审计
-> 统计审计
-> 因子尸检
-> 样本外重测
-> 组合工程审计
-> 执行容量审计
-> 最终判决
```

最终判决只能是以下几类：

- `死亡`：存在无法修复的数据泄露或明显伪 alpha。
- `保留假设`：逻辑有趣，但证据不足。
- `继续研究`：核心信号仍有边际证据，但需要更多测试。
- `进入组合测试`：单独信号通过审计，可以看组合层面。
- `进入模拟实盘`：组合、成本、容量、监控都基本可解释。

不要使用“策略有效”这种模糊结论。

## 4. 深入学习的正确路径

真正的深入不是知道更多术语，而是能不断提出更尖锐的问题。

浅层问题：

```text
这个策略年化多少？
```

中层问题：

```text
这个策略样本外怎么样？
```

深层问题：

```text
在考虑多重检验、标签重叠、风格暴露、交易成本、容量和样本外污染后，它还剩下什么？
```

这个项目所有文档都围绕第三类问题展开。

## 5. 你应该保留的记录

每次审计一个项目，至少保存：

- 原始策略描述。
- 原始回测结果。
- 审计发现。
- 修复尝试。
- 死亡原因或保留原因。
- 未解决问题。
- 下一步实验。

失败策略也要保存。失败记录是你估计 data mining 程度、提升研究判断力的重要材料。
