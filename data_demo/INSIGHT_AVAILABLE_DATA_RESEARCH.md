# 当前 Insight 权限下的数据用途与量化研究路线

本文记录当前账号已经验证可用的数据能力，以及这些数据在量化研究、因子构建、回测和实盘监控中的用法。当前账号更偏“行情系统”权限，适合做高频行情采集、盘口研究、日内因子、交易成本建模和事件驱动回放；不适合直接依赖 Insight 的同步基础数据、财务数据、复权因子和交易日历接口。

## 1. 当前可用与不可用边界

### 已验证可用

实时订阅：

- 股票 tick 快照订阅
- 股票 K 线订阅：秒 K、分钟 K
- 逐笔成交与逐笔委托订阅
- 行情衍生指标订阅
- 新闻订阅
- 融资融券相关订阅
- 期货市场 K 线订阅

行情回放：

- tick 快照回放
- 逐笔成交与逐笔委托回放
- 回放接口支持 `fq='none'`、`fq='pre'`、`fq='post'`

### 当前不可用

同步查询类 `query.get_*` 大多不可用：

- 日 K、分钟 K 同步查询
- 交易日历
- 复权因子
- 股票、债券、基金、指数基础信息
- 财务三表、财务指标
- 公告、研报、新闻同步查询
- 指数成分、ETF 成分
- 两融历史明细同步查询

这意味着研究系统应先围绕“行情流 + 回放”搭建，不要把核心流程依赖在 `query.get_*` 上。

## 2. 最适合先做的系统

### 2.1 实时行情采集器

目标是把可订阅的实时行情保存为本地数据资产。

建议拆成几个进程或模块：

- `collector_realtime_tick.py`：订阅股票 tick 快照
- `collector_realtime_trans_order.py`：订阅逐笔成交和逐笔委托
- `collector_realtime_kline.py`：订阅秒 K、分钟 K
- `collector_realtime_news.py`：订阅新闻事件
- `collector_realtime_margin.py`：订阅两融相关数据
- `collector_realtime_future_kline.py`：订阅期货 K 线

保存格式建议：

- 原始层：Parquet，按日期、市场、数据类型分区
- 快速查询层：DuckDB 或 ClickHouse
- 临时调试层：CSV 或 JSONL

推荐目录结构：

```text
data/insight/raw/tick/trade_date=2026-06-01/market=XSHG/
data/insight/raw/trans_order/trade_date=2026-06-01/market=XSHE/
data/insight/raw/kline_1min/trade_date=2026-06-01/
data/insight/raw/news/trade_date=2026-06-01/
data/insight/raw/margin/trade_date=2026-06-01/
```

保存原则：

- 原始数据尽量不改字段名、不改单位、不做复权
- 每条数据补充本地接收时间 `local_receive_time`
- 每条数据补充采集批次 `collector_run_id`
- 异常、断线、重连事件单独写日志
- 原始数据永远保留，清洗后的数据另存

### 2.2 历史回放采集器

回放接口可以用来补历史，也可以用来测试实时策略。

虽然接口参数支持一个时间区间，但实操建议按天切片：

```text
股票列表 × 单个交易日 × 数据类型
```

这样有几个好处：

- 单次任务更短，失败后容易重跑
- 不容易触发回放数量和时间范围限制
- 文件分区天然按交易日组织
- 后续回测可以逐日加载

回放限制需要注意：

- Tick / Transaction / Order 回放时间范围最多约 30 天
- 存在 `证券只数 × 天数 × 权重` 的乘积限制
- 交易时段回放限制更严格
- 长区间、多股票最好排队分批跑

### 2.3 本地 K 线构建器

当前 `query.get_kline` 不可用，但可以从 tick、逐笔或订阅 K 线构建自己的 K 线。

可构建频率：

- 1 秒 K
- 5 秒 K
- 15 秒 K
- 1 分钟 K
- 5 分钟 K
- 15 分钟 K
- 日内 session K
- 日线近似 K

字段建议：

- `open`
- `high`
- `low`
- `close`
- `volume`
- `amount`
- `trade_count`
- `vwap`
- `buy_volume`
- `sell_volume`
- `buy_amount`
- `sell_amount`
- `large_buy_amount`
- `large_sell_amount`
- `turnover_proxy`
- `first_tick_time`
- `last_tick_time`

如果使用逐笔成交构造 K 线，质量会比单纯用快照更好。快照适合构造行情状态，逐笔成交更适合构造成交统计。

## 3. 可以构建的因子

### 3.1 价格与收益率因子

基于 tick 或分钟 K：

- 日内动量：过去 1 分钟、5 分钟、15 分钟收益率
- 开盘动量：开盘后 5 分钟、15 分钟涨跌幅
- 尾盘动量：14:30 到收盘收益率
- VWAP 偏离：当前价相对当日 VWAP 的偏离
- 高低点突破：当前价是否突破过去 N 分钟高点
- 回撤修复：从日内低点反弹幅度
- 区间反转：短期快速上涨后回落概率

适合用途：

- 日内择时
- 自选股监控
- 超短线信号
- 日内横截面排序

注意事项：

- A 股 T+1 交易限制会影响信号可交易性
- 股票涨跌停会让价格动量失真
- 开盘集合竞价后的短窗口噪音很大，需要单独处理

### 3.2 成交量与成交额因子

基于 tick、逐笔成交、分钟 K：

- 成交量放大倍数：当前分钟成交量 / 过去 N 日同分钟均值
- 成交额冲击：最近 N 秒成交额占当日成交额比例
- 大单成交占比：大单金额 / 总成交金额
- 小单活跃度：小额成交笔数占比
- 成交密度：单位时间成交笔数
- 量价配合：上涨时放量、下跌时缩量等结构
- 异常成交额排名：横截面对比同一时刻成交额分位数

适合用途：

- 放量突破监控
- 大资金异动识别
- 日内流动性筛选
- 交易容量评估

### 3.3 主动买卖压力因子

基于逐笔成交方向或价格变动推断：

- 主动买入金额
- 主动卖出金额
- 净主动买入金额
- 主动买入笔数占比
- 主动买入金额占比
- 大单主动买入占比
- 连续主动买入强度
- 主动买入衰减速度

可构造：

```text
net_buy_pressure = active_buy_amount - active_sell_amount
buy_pressure_ratio = active_buy_amount / (active_buy_amount + active_sell_amount)
large_buy_pressure = large_active_buy_amount - large_active_sell_amount
```

适合用途：

- 捕捉短期资金推动
- 识别拉升、出货、对倒行为
- 做日内横截面强弱排序

注意事项：

- 买卖方向如果不是交易所原生字段，而是通过 tick rule 推断，会有误差
- 大单阈值不能固定，应按股票流动性分层

### 3.4 盘口与订单簿因子

如果 tick 快照里有买卖盘档位，可以构造：

- 一档价差
- 五档价差
- 买一卖一挂单量差
- 五档委托量不平衡
- 买盘厚度
- 卖盘厚度
- 委托簿斜率
- 盘口撤单后的价格反应
- 排队压力

常见形式：

```text
obi_1 = (bid_size_1 - ask_size_1) / (bid_size_1 + ask_size_1)
obi_5 = (sum_bid_size_1_5 - sum_ask_size_1_5) / (sum_bid_size_1_5 + sum_ask_size_1_5)
spread = ask_price_1 - bid_price_1
spread_bps = spread / mid_price * 10000
```

适合用途：

- 高频择时
- 交易执行优化
- 判断短期冲击成本
- 策略下单前过滤流动性差的标的

### 3.5 波动率与流动性因子

可构造：

- Realized volatility
- Parkinson 波动率
- 日内振幅
- 分钟收益标准差
- Amihud illiquidity
- 成交额调整后的价格冲击
- bid-ask spread 均值
- 零成交分钟占比
- 单笔成交金额中位数

适合用途：

- 风险控制
- 仓位缩放
- 流动性过滤
- 交易成本模型

### 3.6 逐笔委托与撤单因子

如果逐笔委托数据字段足够完整，可以研究：

- 新增买单强度
- 新增卖单强度
- 撤买单强度
- 撤卖单强度
- 大额委托出现频率
- 大额委托撤单率
- 订单簿补单速度
- 委托方向失衡

可用于：

- 判断盘口真假强弱
- 识别诱多/诱空
- 优化挂单/吃单决策
- 构建微观结构 alpha

### 3.7 新闻与事件因子

新闻订阅可用于事件驱动框架：

- 股票相关新闻出现次数
- 新闻发布时间距离开盘/收盘的距离
- 标题关键词分类
- 情绪模型打分
- 新闻后 1 分钟、5 分钟、30 分钟收益
- 新闻后成交量放大倍数

当前如果没有同步新闻历史查询权限，实时新闻适合做：

- 实时事件监控
- 事件样本逐步积累
- 未来构建新闻事件库

### 3.8 两融相关因子

两融订阅可用于：

- 融资融券标的池变化监控
- 融券可借状态监控
- 标的资格变化事件
- 与成交活跃度结合，构建做空约束 proxy

如果后续能补到两融历史明细，可进一步构造：

- 融资余额变化
- 融券余额变化
- 融资买入额占成交额比例
- 融券卖出额占成交额比例

## 4. 回测框架建议

### 4.1 两层回测

建议拆成两层，而不是一开始就写一个很重的系统。

第一层：因子研究回测。

- 使用本地聚合的分钟线或日内特征
- 做横截面排序、分组回测、IC、RankIC
- 重点验证信号是否有预测力
- 成交模拟可以先简化

第二层：事件驱动回放回测。

- 使用 Insight 回放或本地 tick 数据逐条推进
- 模拟策略在真实时间序列中的触发
- 加入滑点、成交量限制、涨跌停、停牌、不可成交等约束
- 验证策略从信号到交易的完整链路

### 4.2 回测数据选择

原始价格和复权价格要分清：

- 成交模拟：用 `fq='none'`
- 日内策略：通常用 `fq='none'`
- 长周期趋势和收益率研究：优先用前复权
- 多年日线回测：需要复权因子或前复权价格

当前 Insight 回放接口支持 `fq` 参数：

```python
fq = "none"  # 原始价格
fq = "pre"   # 前复权
fq = "post"  # 后复权
```

但建议原始落库永远保存 `fq='none'`，研究衍生数据再按需要生成前复权版本。原始成交价格是最接近真实交易的基准。

### 4.3 交易成本建模

当前权限非常适合做交易成本模型：

- bid-ask spread
- 成交额容量
- 冲击成本
- VWAP 偏离
- 下单后 N 秒价格漂移
- 盘口深度
- 大单成交对价格影响

可以按这些维度分组：

- 市值分层
- 成交额分层
- 波动率分层
- 行业分层
- 时间段分层
- 涨跌停状态

交易成本模型可以反过来服务日线策略：即使 alpha 来自日线，实际调仓也需要估算冲击成本和容量。

## 5. 数据质量与风控要点

### 5.1 交易日历缺失

当前 `query.get_trading_days` 不可用，因此需要外部交易日历：

- akshare
- exchange_calendars
- baostock
- 自己维护 CSV
- 交易所公告源

交易日历至少要支持：

- 是否交易日
- 开盘/收盘时间
- 午休时间
- 半日市
- 节假日
- 特殊停市

### 5.2 复权因子缺失

如果只做日内交易，复权不是最紧急的问题。如果做跨天或多年回测，需要补复权因子：

- Tushare
- JoinQuant
- AkShare
- RiceQuant
- Wind/Choice/同花顺等商业源
- 自己从分红送转公告计算

没有复权因子时，跨除权日的收益率、均线、突破信号都会失真。

### 5.3 股票基础信息缺失

需要外部补充：

- 股票上市日期
- 退市日期
- ST 状态
- 涨跌停比例
- 证券简称
- 行业分类
- 市值
- 流通股本
- 指数成分

否则容易出现：

- 幸存者偏差
- 未来函数
- 错误股票池
- 对新股、退市股处理不当

### 5.4 时间戳处理

高频数据最容易出问题的是时间：

- 行情时间 `MDTime`
- 本地接收时间 `local_receive_time`
- 交易所撮合时间
- 回放推送时间
- Python 处理时间

建议全部保留，不要只存一个时间字段。

### 5.5 断线与缺口

实时采集必须记录：

- 登录成功时间
- 订阅成功时间
- 断线时间
- 重连时间
- 每个标的最后一条数据时间
- 数据静默超时

后续研究因子时，要过滤采集不完整的区间。

## 6. 推荐建设顺序

### 第一阶段：把数据稳定存下来

目标：

- 保存实时 tick
- 保存实时逐笔成交/委托
- 保存回放 tick
- 保存回放逐笔成交/委托
- 建立统一 Parquet 分区

验收标准：

- 任意一天、任意股票可以加载完整 tick
- 可以统计每只股票每日数据条数
- 可以识别缺失分钟
- 可以重跑某一天某只股票

### 第二阶段：构造分钟特征

目标：

- 从 tick/逐笔构造 1 分钟 K
- 计算成交量、成交额、VWAP、买卖压力
- 生成日内特征表

验收标准：

- 每只股票每天有完整分钟 bar
- 能画出价格、成交量、买卖压力曲线
- 能按分钟做横截面排序

### 第三阶段：做因子检验

目标：

- 计算 RankIC
- 分组收益
- 多空组合
- 分时段表现
- 分流动性表现

优先检验：

- 短期动量/反转
- 主动买卖压力
- 放量异动
- 盘口不平衡
- VWAP 偏离
- 流动性冲击

### 第四阶段：做事件驱动回测

目标：

- 用本地 tick 或 Insight 回放驱动策略
- 模拟信号触发
- 加入成交约束
- 评估滑点和容量

验收标准：

- 同一策略在回放和本地数据上结果一致
- 下单时间不使用未来数据
- 交易成本可配置
- 支持逐日批量回测

### 第五阶段：接入外部基础数据

目标：

- 补交易日历
- 补复权因子
- 补股票基础信息
- 补行业、市值、指数成分

这一步完成后，才适合做更严肃的中低频股票多因子回测。

## 7. 适合当前权限的研究题目

优先级较高：

- A 股日内成交量异动与短期收益
- 主动买卖压力对未来 1 到 30 分钟收益的预测
- 盘口不平衡与短期价格冲击
- 大单成交后的价格延续或反转
- 开盘 15 分钟资金流对全天收益的预测
- 尾盘成交结构与次日开盘收益
- VWAP 偏离后的均值回归
- 不同流动性股票的交易成本模型
- 股票涨停前的盘口和逐笔特征
- 新闻订阅事件后的短期行情反应

暂时不优先：

- 基于财务报表的基本面因子
- 多年日线价值因子
- 指数成分历史回测
- 需要精准复权和完整公司行为的长周期策略
- 依赖公告、研报历史库的大样本事件研究

## 8. 最小可行研究闭环

一个实用闭环可以这样搭：

```text
Insight 实时/回放
  -> 原始 Parquet
  -> 清洗标准化
  -> 1min/5min bar
  -> 日内因子
  -> 因子检验
  -> 事件驱动回放
  -> 策略报告
```

这个闭环不依赖当前不可用的 `query.get_*`，但能产出有研究价值的微观结构和日内 alpha。后续只要补齐交易日历、复权因子和股票基础信息，就可以自然扩展到完整多因子研究平台。
