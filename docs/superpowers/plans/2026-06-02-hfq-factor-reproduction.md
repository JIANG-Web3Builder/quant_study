默认口径已锁定：

- 数据：多源并行；Insight 负责高频行情，AkShare/Tushare 等补交易日历、股票池、指数成分、复权和基础信息。

- 区间：第一轮复现为 `2018-01-08` 至 `2023-12-01`。
- 资源：Windows 单机扩展，Parquet 分区 + DuckDB 查询，不默认使用集群。
- 交付：以复现报告为主线，但工程模块必须可复用。

## Key Changes

1. 建立工程骨架
   在 `D:\quant_study\src\hfq_repro` 下创建数据、因子、回测、评估、报告模块；在 `D:\quant_study\configs\hfq_repro.yaml` 固化日期区间、数据源优先级、股票池、分区路径、批大小和并发数。

2. 统一数据接口
   定义这些核心表：
   - `raw_tick`：Insight tick/快照原始数据。
   - `raw_trans_order`：逐笔成交和逐笔委托。
   - `bar_1m`、`bar_3s`、`bar_30s`：统一K线/快照派生层。
   - `stock_universe_daily`：每日可交易股票池、指数成分、ST/停牌/上市退市状态。
   - `factor_daily`：`trade_date, stock_code, factor_code, value, source_version`。
   - `factor_eval_metrics`：IC、RankIC、ICIR、分组收益、多空收益、超额收益、换手率、夏普、最大回撤、胜率。

3. 数据采集与治理
   Insight 高数据量任务按 `trade_date x market x data_type` 分片，写入 `D:\quant_study\data\hfq\raw\...`；每个分片有 manifest，记录状态、行数、时间范围、数据源、错误和重跑次数。
   外部源仅补 Insight 当前缺失的同步查询能力：交易日历、全市场股票列表、指数成分、复权因子、ST/停牌/上市退市、行业和市值。

4. 因子构建
   以 `factor_specs.csv` 为注册表，先实现 16 个 `top` 因子和 25 个 `high` 因子，再补齐全部 87 个。
   后缀口径按 PDF OCR 锁定：
   - `_o`：最后一期有效值。
   - `_m`：日内均值。
   - `_std`：日内标准差。
   - `_z`：日内标准分。
   复合因子按报告方式：每类取周度综合表现前5名，做市值和行业中性化、标准化后等权合成。

5. 回测与评估
   支持月度和周度调仓，股票池包括全市场、沪深300、中证500、中证1000。
   基准为有效股票池等权收益；输出 PDF 对应图表：有效性表、分组收益、多空净值、多头超额净值、相关性矩阵和最终复现报告。

## Implementation Plan

1. 复现口径审计
   校验 `factor_specs.csv`、`factor_replication_map.csv`、`backtest_chart_map.csv` 与 PDF/OCR 是否一致，生成一份 `factor_manifest`，标记每个因子所需数据、频率、公式、变体、优先级和对应图表。

2. 环境补齐
   在现有 `.venv` 中补安装并固定依赖：`pyarrow`、`duckdb`、`polars`、`scipy`、`scikit-learn`、`statsmodels`、`akshare`、`tushare`。
   统一 UTF-8 读写，避免现有 Markdown 在 PowerShell 下乱码影响后续报告。

3. 数据层实现
   先实现 source adapter：
   - `InsightAdapter`：复用现有 `data_demo` 登录和回放能力。
   - `AkshareAdapter` / `TushareAdapter`：补股票池、指数成分、交易日历和复权。
   - `DataQualityChecker`：检查缺分钟、重复、时间戳异常、零成交、涨跌停和停牌。

4. 全市场数据采集
   生产目标直接跑全市场 `2018-01-08` 到 `2023-12-01`，但每个采集任务都必须可独立重跑。
   优先级：1分钟可复现因子所需数据先落库，再补 tick 深度、3s/30s 快照、逐笔成交/委托，以便逐步覆盖拥挤度、盘口、资金流类因子。

5. 因子计算
   按类别实现：
   - 动量反转、波动率、高阶特征、流动性、量价相关性、筹码分布。
   - 再实现拥挤度、资金流。
   - 最后实现复合因子。
   每个因子输出统一进入 `factor_daily`，保留原始值、变体值和方向信息。

6. 指标与回测
   实现 IC/RankIC、ICIR、分组收益、多空收益、多头超额、换手率、夏普、最大回撤、胜率。
   月度/周度调仓都跑；每个股票池单独产出结果，并与 PDF 的关键指标做误差对比。

7. 报告生成
   生成 `D:\quant_study\reports\hfq_repro\` 下的复现报告，包含：
   - 数据覆盖率和质量审计。
   - 87 个因子的指标总表。
   - 每类 top 因子的图表复现。
   - 与 PDF 结论一致/不一致的解释。
   - 最佳因子筛选和复合因子回测结果。

8. 超越报告部分
   在完成 PDF 对齐后追加：
   - 样本外滚动检验。
   - 分市值、行业、流动性分层。
   - 交易成本和容量压力测试。
   - 因子相关性和去冗余选因子。
   - 自动挑选最优单因子/复合因子进入组合回测。

## Test Plan

- 单元测试：每个因子用小型人工数据验证公式、后缀变体、空值处理和方向。
- 数据测试：每个交易日检查全市场覆盖率、分钟完整性、重复记录、异常价格、异常成交量。
- 回测测试：用固定小样本验证 IC、分组收益、多空净值、换手率和调仓日逻辑。
- 复现验收：16 个 `top` 因子关键指标方向必须与 PDF 一致；ICIR、多空收益、单调性差异需要在报告中解释。
- 工程验收：任意失败分片可重跑；任意一个因子可单独重算；任意股票池和日期区间可单独出报告。

## Assumptions

- 当前 Insight 账号可持续访问回放 tick 和逐笔成交/委托；若某类历史高频数据权限不足，该类因子先标记为 blocked，并用可用频率近似复现。
- AkShare/Tushare 的外部基础数据允许用于研究补全；若 Tushare token 不可用，先用 AkShare，但在报告中标记数据源差异。
- 第一轮严格按 `2018-01-08` 至 `2023-12-01`；PDF 中更早的 `2013-2023` 结果不作为第一轮硬验收。
- 单机存储不足时，不改变研究口径，只调整采集顺序和分片批量。
