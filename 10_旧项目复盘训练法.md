# 旧项目复盘训练法

## 1. 为什么旧项目是最好的训练材料

你已经做过很多项目，这比单纯看教程更有价值。真正的提升来自：

```text
把自己过去相信的东西重新审判一遍。
```

旧项目有三个优点：

- 你熟悉上下文。
- 你知道当时怎么调出来的。
- 你能看到自己过去的研究习惯。

进阶训练不是证明过去的自己很强，而是找出过去的自己哪里不够严谨。

## 2. 复盘对象选择

优先选择：

- 你最满意的策略。
- 收益曲线最漂亮的策略。
- 使用 ML 的策略。
- 高频换手策略。
- 基本面因子策略。
- 小市值表现很强的策略。
- 样本外一般但样本内很好的策略。

不要只挑弱策略。最值得审计的是你最相信的策略。

## 3. 三层复盘

### 第一层：事实复盘

记录当时真实情况：

```text
我当时试了多少版本？
我为什么选择最终版本？
我有没有看过测试集后修改？
我有没有记录失败实验？
我用了哪些数据？
我当时最担心什么？
```

不要美化。

### 第二层：红队复盘

按本文档体系攻击：

- 数据泄露。
- 统计显著性。
- 因子尸检。
- 样本外污染。
- 组合工程。
- 成本容量。
- ML 反模式。

目标是找死亡原因或保留证据。

### 第三层：能力复盘

问自己：

```text
这个项目暴露了我哪类能力缺口？
```

常见缺口：

- 数据纪律不足。
- 统计推断不足。
- 风险暴露理解不足。
- 执行成本估计不足。
- 样本外设计不足。
- 研究记录不足。
- 解释能力不足。

## 4. 项目判决分类

### A 类：死亡项目

特征：

- 数据泄露。
- 样本外反向。
- 成本后消失。
- 完全来自小市值或低流动性。
- 参数孤岛。

处理：

```text
记录死亡原因，不再投入。
```

### B 类：假设保留项目

特征：

- 逻辑有趣。
- 当前证据不足。
- 样本外弱。
- 需要更干净数据或更保守验证。

处理：

```text
保留研究假设，重新设计实验。
```

### C 类：可继续研究项目

特征：

- 红队后仍有残余效果。
- 风险暴露可以解释。
- 成本后仍有一点空间。
- 样本外没有反向。

处理：

```text
进入组合测试或模拟实盘。
```

## 5. 每周训练计划

每周复盘一个旧项目。

输出一份 2-4 页审计：

```text
第 1 页：原始策略与原始结果
第 2 页：红队攻击发现
第 3 页：关键补充实验
第 4 页：最终判决与能力缺口
```

12 周后，你会得到：

- 12 个项目死亡/保留/继续研究判决。
- 一份自己的常见错误清单。
- 一套真实项目训练材料。
- 比泛泛看书更清晰的能力地图。

## 6. 复盘问题清单

每个项目都问：

- 如果这个策略是假的，最可能假在哪里？
- 哪个实验最可能杀死它？
- 它是否依赖某个市场阶段？
- 它是否依赖某个股票池？
- 它是否依赖低成本假设？
- 它是否依赖小市值或低流动性？
- 它的收益是否集中在少数股票？
- 它的测试集是否被污染？
- 如果今天重新做，我会怎样设计？

## 7. 最终目标

旧项目复盘的最终目标不是留下很多策略，而是提升研究判断力。

理想结果可能是：

```text
12 个旧项目里，8 个死亡，3 个保留假设，1 个值得继续研究。
```

这不是失败。恰恰相反，这说明你的研究标准变高了。

普通人害怕策略死亡。成熟研究员感谢策略尽早死亡。
